**original_problem_statement:**
Build a cross-platform, offline-first application for a Farmer Producer Company (FPO) to manage inventory, sales, procurement, and billing.

**PRODUCT REQUIREMENTS (based on initial request and subsequent user feedback):**
- **User Roles:** Admin, Agent, Farmer, Shareholder.
- **Authentication:** Secure login. The main issue has been intermittent login failures on the deployed APK.
- **Admin Management:** Full CRUD for Products, Outlets, and Vendors. Approve agent registrations, shareholder upgrades, and stock transfers (including partial/modified approvals).
- **Procurement:** A unified procurement screen for recording purchases from both Farmers and Vendors. Must support searching for existing entities and also allow manual entry of names for quick transactions. Procurement must be allocated to an outlet.
- **Stock Management:** Agents can request stock transfers. Admins can view and approve/reject/modify these requests.
- **Sales & Billing:** Record sales. Search for products, outlets, and customers. Allow manual entry of customer names for quick transactions.
- **Farmer/Agent Interaction:** Farmers can raise buy/sell requests for a specific outlet. Agents and Admins get notified. Farmers can view their transaction history.
- **Shareholder Flow:** Farmers can apply to become shareholders by uploading a certificate. Admins must have a UI to approve/reject these applications.
- **Notification System:** A dedicated notification tab for all roles to see relevant events (requests, approvals, transfers).
- **Reporting:** Role-specific reports (Sales, Stock, etc.) with mandatory date-range filtering, exportable to Excel.
- **Search:** Global search functionality for products, outlets, farmers, and vendors across relevant screens (sales, procurement, stock).

**User's preferred language**: English

**what currently exists?**
The application has a FastAPI backend and an Expo (React Native) frontend. A significant number of features have been coded in this session, including:
- Auth flow refactoring with enhanced logging.

- Frontend screens and UI elements for most of these features, including a unified procurement screen, an admin stock management screen with an approval flow, and report filtering.
