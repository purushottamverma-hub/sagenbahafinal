**original_problem_statement:**
Build a cross-platform, offline-first application for a Farmer Producer Company (FPO) to manage inventory, sales, procurement, and billing.

**PRODUCT REQUIREMENTS (based on initial request and subsequent user feedback):**
- **User Roles:** Admin (full access), Agent (outlet-level access, can also be a farmer), Farmer (can view their data, make requests), Shareholder (upgraded farmer).
- **Authentication:** Secure login. Admins manage all users. Users can self-register as Farmers or Agents. Agent registration requires admin approval.
- **User Management (Admin):** Admins can add/delete/edit Products, Outlets, and Vendors. They can view all users, approve pending agents, and upgrade farmers to shareholders.
- **Inventory Management:** Track stock per product/outlet. Admin can transfer stock between outlets. Admin and Agents can add stock to outlets. Stock should be auto-deducted upon sale.
- **Sales & Billing:** Record sales (cash, online, credit). Generate and share/print PDF bills.
- **Procurement:**
    - **From Farmers:** Record produce purchases.
    - **From Vendors:** Admin can register vendors and record purchases from them.
- **Farmer & Agent Interaction:**
    - Farmers can raise buy/sell requests, selecting a target outlet and product (or entering a custom one).
    - These requests should appear as notifications/pending tasks for the relevant agent and admin.
    - The request can be marked as resolved by any party involved.
- **Reporting:** Generate and export reports (Sales, Stock, Customer, Farmer, Purchase) to Excel, with date-range filtering.
- **Technical:** PWA for web/desktop and an Android app. Bilingual (Hindi/English).

**User's preferred language**: English (Application UI is bilingual English/Hindi)

**what currently exists?**
The application is in an advanced state. The backend (FastAPI) has extensive models and API endpoints for auth, users, products, sales, stock, farmer procurement, and initial versions of vendor and report management. The frontend (Expo) has a robust, role-based navigation system for Admin, Agent, and Farmer users. Key features like user login/signup, state persistence (via a custom manual implementation), sales recording, and multi-tab dashboards for all roles are functional. A reports page with Excel export buttons and a comprehensive user management screen have been created.
